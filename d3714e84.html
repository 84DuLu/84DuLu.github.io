CS50 的 make 和 Makefile 简单介绍

CS50 中的 make

在 CS50 中，我们是如何来编译并运行可执行文件的呢？假如，当前目录下有一个 hello.c 的文件，我们可以根据 CS50 的教程，用 make hello 来生成可执行文件，然后再用 ./hello 来运行这个可执行文件。

在 week02 的学习中，我们知道 cs50 的 make 是利用 GNU 下的 make 工具，调用 clang 编译器来把 source code 转换成 machine code 的。但仅仅知道这些，可能满足不了大家的好奇心，我们来看看它是怎么做到的。

printenv

我们用这个命令打印出环境变量。

我们可以发现环境变量中有这么两行，CC 指示出了 make 所要用的编译器，而 CFLAGS 表示 CC 要用的的一些参数。指定了这两个东东（它们是特殊变量），make 自带的一些规则就可以对 .c 文件进行编译并生成可执行文件。我们可以用以下命令达到相同的效果（只是模仿）：

$CC $CFLAGS hello.c -o hello

我们应该大概能懂 cs50 中的 make 对单个 c 文件进行处理是什么样的过程了。

编译器 gcc 的简单使用

有些朋友可能会对上面的过程有点懵，我们来看一下如果不用 make，只用编译器 gcc （或clang）该怎么编译文件。我这里用 gcc 演示啦，因为不太了解 clang，不过大体的一些东西应该差不多。平常我们想要编译一个文件，大概是这样的过程。

如果我们用 gcc hello.c 这样的命令来编译文件，它不仅会完成编译的工作，还会启动连接器完成链接的工作。

如果我们只想让他编译成目标文件，我们可以添加 -c 的参数，这样就不会启动连接器了。

gcc -c hello.c

我们现在修改下 hello.c 的代码

我们这时候再用 gcc hello.c 就不行了，因为 CS50 不是 C 的标准库，所以我们要通过 -l 的参数把它连接件进来。

这里 -o hello 是把它输出的可执行文件的名字改成 hello 。

当我们引入了 cs50 库时，其实我们上面模仿 make 隐式规则的命令也失效了，怎么来改正呢？通过查阅文档可以发现，隐式规则所使用的变量 LDLIBS 与链接库有关。

再次用 printenv 查看 cs50 为我们设置好的环境变量可以发现，正好是有这一项的。

把命令改成 $CC $CFLAGS hello.c -o hello $LDLIBS ，我们就又粗略地模拟出来 make 的过程了。

Makefile 的编写

The make utility automatically determines which pieces of a large program need to be recompiled, and issues commands to recompile them.（摘自 GNU make 文档）

好像前面说的有点多了，接下来让我们看看简单的 Makefile 应该怎么编写吧。尽管我们可以用 gcc hello.c 来生成单个 C 文件的可执行文件，不过我们还是应该看看编译单个 C 语言文件的 Makefile 应该怎么写。

Makefile 的命名

让我们来试着写一个 Makefile 出来吧。首先，要起一个合适的名字。它可以是 Makefile、makefile 和 GNUmakefile 。最后一种是不推荐的，因为它只能被 GNU 的 make 识别。除此之外你也可以起点别的名字，但是你 make 的时候要加参数 -f 。
